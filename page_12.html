Cons of Using Rust on the Host Side

Steeper Learning Curve

Requires understanding ownership, borrowing, and lifetimes — concepts unfamiliar to C/C++ developers

Longer Compile Times

Incremental build overhead and slower full rebuilds can impact development velocity

Smaller Low-Level Hardware Ecosystem

Fewer mature libraries and tools for specialized hardware interfaces and legacy protocols

FFI Complexity with Existing C Code

Interfacing with C libraries requires unsafe blocks and manual memory management at boundaries

Not Ideal for Ultra-Low-Level Firmware

Potential binary size overhead and toolchain friction in highly constrained embedded environments
